# 文章润色与封面生成工具 - 产品需求文档（PRD）

## TL;DR
构建一个Web应用，帮助用户完整完成文章润色→生成封面的工作流。使用两个豆包模型（文本润色 + 图像生成），直接下载API结果无需本地存储，支持拖拽上传、双向编辑、灵活导出。

---

## 完整工作流

### 第一步：初稿输入与润色
1. 用户输入markdown格式的初稿（**支持方式**）：
   - 直接粘贴到文本框
   - **拖拽上传md文件**（新增）到输入框自动读取
2. 初稿框**支持实时编辑**（新增功能），用户可随时修改内容
3. 输入润色要求（可选）
4. 点击"润色"按钮 → API调用豆包文本模型
5. 显示润色结果（关键改进：同时显示两个版本进行对比）
   - 初稿显示在左侧（可继续编辑）
   - 润色结果显示在右侧（**支持实时编辑**，新增功能）
6. 如不满意，可修改润色要求后重新润色，覆盖当前版本

### 第二步：封面生成
1. 点击"生成封面"按钮，进入封面生成页面
2. **页面左侧**：Prompt输入框（支持实时编辑）
   - 默认显示图片尺寸提示："192*128px"
   - 用户可修改此提示信息或替换为自定义prompt
3. **页面右侧**：实时预览生成的4张图片（或占位符）
4. 用户确认prompt后，点击"生成"按钮
5. 调用豆包图像生成模型 → 返回4张图片
6. 四张图在右侧全显示，用户可选择下载单张或点击"重新生成"修改prompt后重新生成

### 第三步：文件导出
1. **导出润色后的文章**：
   - 支持导出为markdown文件（下载）
   - **支持"另存为"功能**（新增），可自定义文件名
2. **导出图片**：
   - 可分别下载各张图片

## 关键补充/改进建议

### 1. 界面结构改进
- **分屏对比视图**：左侧显示初稿/当前版本，右侧显示新润色结果，便于对比
- **双向可编辑**：初稿框和润色后框都支持实时编辑（新增）
- **拖拽上传**：支持拖拽md文件到输入框快速导入（新增）
- **进度指示**：显示当前在工作流的哪个阶段（输入 → 润色 → 审核 → 生成封面）
- **富文本编辑**：markdown实时预览，而非纯文本框


### 2. 提示词（Prompt）管理
- **预设模板库**：针对不同文章类型有预设封面生成prompt
- **可自定义参数**：生成时可调整"风格"（简约/现代/复古）、"主色调"等
- **动态prompt生成**：根据文章标题/摘要自动生成初始prompt
- **Prompt本地缓存**（可选）：保存用户常用的prompt到本地，便于快速复用

### 3. 安全性与配置
- **.env配置**：存储豆包API密钥
- **前端隐私**：不在源码中暴露密钥
- **本地优先**：优先使用本地机制存储数据，减少依赖后端
- **API调用限制**：显示当月API使用额度（如果有限制）

### 4. 用户体验优化
- **快捷键支持**：
  - Ctrl+E 快速润色
  - Ctrl+G 快速生成封面
  - Ctrl+S 快速导出
- **暗黑模式**：支持系统主题切换

### 5. 错误处理与反馈
- **API超时处理**：显示超时提示，支持重试
- **不同错误的友好提示**：网络错误、API配额不足、无效密钥等
- **请求进度条**：显示API调用进度（特别是图片生成，会较慢）
- **使用统计面板**：显示今日已使用token、剩余额度等

## 数据流与时序（简化版 - 无本地存储）

```
用户输入初稿（拖拽上传 或 粘贴文本）
          ↓
输入润色要求 + 点击"润色"
          ↓
调用豆包API → 获取润色结果
          ↓
显示分屏对比（左侧初稿，右侧润色结果）
          ↓
用户可实时编辑两侧内容
          ↓
不满意？ ← 修改要求后重新润色
满意？   → 继续下一步
          ↓
点击"生成封面" → 确认prompt
          ↓
调用豆包图像API并发生成4张图
          ↓
显示4张图片（用户预览和选择）
          ↓
点击"下载" → 直接下载到本地
          ↓
✓ 完成（无本地存储占用）
```

## 补充说明

### API接口配置指引（.env文件）

用户需要在`.env`文件中配置豆包API接口，示例：

```env
# 文本润色API配置
VITE_TEXT_POLISH_API_URL=https://your-api-endpoint/v1/text/polish
VITE_TEXT_POLISH_API_KEY=your_api_key_here

# 图像生成API配置
VITE_IMAGE_GEN_API_URL=https://your-api-endpoint/v1/images/generate
VITE_IMAGE_GEN_API_KEY=your_image_api_key_here

# 模型参数配置（可选）
VITE_TEXT_MODEL_NAME=doubao-pro
VITE_IMAGE_MODEL_NAME=doubao-image-gen
VITE_TEMPERATURE=0.7
VITE_TOP_P=0.9
```

**配置说明**：
- 用户根据自己的豆包账户信息，在`.env`中填写实际的API接口地址和密钥
- 前端通过环境变量读取这些配置，调用豆包API
- `.env`文件不会提交到git（`.gitignore`中已配置）
